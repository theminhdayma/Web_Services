So sánh giữa SOAP và REST

Tiêu chí SOAP REST
Kiến trúc Chủ yếu là stateful, nhưng có thể stateless Stateless (không lưu trạng thái giữa các request)
Định dạng dữ liệu Chỉ hỗ trợ XML Hỗ trợ nhiều định dạng: JSON, XML, HTML, v.v.
Tính linh hoạt Chỉ sử dụng HTTP POST Sử dụng đầy đủ các phương thức HTTP: GET, POST, PUT, DELETE
Tính bảo mật Hỗ trợ WS-Security, thường dùng HTTPS Dựa vào HTTPS, có thể tích hợp OAuth, JWT
