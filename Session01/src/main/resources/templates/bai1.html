Bảng So Sánh Spring MVC và Spring Boot

Tiêu chí Spring MVC Spring Boot
Khái niệm Là module của Spring Framework, dùng để xây dựng ứng dụng web theo mô hình MVC truyền thống. Là framework mở rộng dựa trên Spring, giúp phát triển ứng dụng Spring nhanh chóng với cấu hình tối thiểu.
Cấu hình Cần cấu hình thủ công nhiều file XML hoặc JavaConfig, phải tự cấu hình DispatcherServlet, ViewResolver, DataSource, v.v. Hầu như không cần cấu hình thủ công, sử dụng auto-configuration, chỉ cần cấu hình những gì thực sự đặc biệt.
Khởi tạo dự án Thường phải tạo project từ đầu, tự thêm các dependency cần thiết, cấu trúc thư mục thủ công. Có thể tạo nhanh qua Spring Initializr, project mẫu tự động sinh ra đầy đủ cấu trúc và dependency cơ bản.
Quản lý phụ thuộc Phải tự thêm và quản lý version các thư viện trong file build, dễ gặp lỗi xung đột version. Được quản lý tập trung qua Spring Boot Starter, tự động chọn version phù hợp, tránh xung đột.
Thời gian phát triển Tốn nhiều thời gian cho việc cấu hình, setup ban đầu và xử lý lỗi cấu hình. Rút ngắn thời gian phát triển nhờ cấu hình tự động, dễ dàng mở rộng và tích hợp.
Triển khai & chạy Cần build war và triển khai lên server (Tomcat, Jetty, ...). Có thể chạy trực tiếp với java -jar, tích hợp sẵn server nhúng.
Hỗ trợ microservices Không tối ưu cho microservices, chủ yếu dùng cho ứng dụng đơn lẻ (monolithic). Hỗ trợ mạnh mẽ cho microservices, tích hợp tốt với Spring Cloud, Docker, Kubernetes,...
